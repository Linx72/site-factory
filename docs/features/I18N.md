# i18n (next-intl)

Optional multi-locale routing for marketing pages.

## Enable

```bash
# .env.local
NEXT_PUBLIC_I18N=true
```

Restart dev server. Root `/` redirects to `/en`. Russian copy at `/ru`.

## Files

| Path | Purpose |
|------|---------|
| `src/i18n/routing.ts` | Locales: `en`, `ru` |
| `src/i18n/request.ts` | next-intl request config |

| `messages/en.json`, `messages/ru.json` | Hero, stats, timeline, testimonials, team, features, pricing, FAQ, contact, CTA, header |
| `src/lib/i18n/section-copy.ts` | Types + FAQ ids |
| `src/lib/i18n/build-home-copy.ts` | Server helper for `[locale]/page.tsx` |
| `src/app/[locale]/page.tsx` | Localized home |
| `src/app/[locale]/motion/page.tsx` | Localized motion lab |
| `src/app/[locale]/status/page.tsx` | Localized status dashboard |
| `src/lib/i18n/alternate-links.ts` | hreflang + canonical for sitemap and metadata |
| `src/lib/i18n/paths.ts` | `localizedPath`, `localizedHomeHref` |
| `src/middleware.ts` | No-op when i18n off |

## Adding strings

1. Add keys to `messages/en.json` and `messages/ru.json`
2. Use `getTranslations('Home')` in server components
3. Call `buildHomeSectionCopy(t => t(key))` and pass `copy` to `HomePage`
4. Hero props still passed separately (see `[locale]/page.tsx`)

Localized sections: **all home sections** including **UI showcase**, **scroll story title**, **pricing price labels**, **checkout buttons**, plus **Nav**, **Footer**, **header** (CTA + menu).

Secondary routes when i18n is on: **`/en/motion`**, **`/ru/motion`**, **`/en/status`**, **`/ru/status`** with localized Convex dashboard panels (seed buttons, CMS editor labels) and **motion lab demo labels**.

**404 and checkout:** `NotFound` and `Checkout` namespaces in `messages/*.json` — localized 404 page and Stripe success banner (`?checkout=success`).

**Hero stats:** three stat cards under the hero (`hero.stats.*`) are localized on `/ru`.

**A11y:** `A11y` namespace — skip link (`SkipToContent`), theme toggle aria labels, logo marquee `aria-label` from `logos.title`.

**Metadata:** `/[locale]` home and status pages use translated `title` / `description` from messages. Twitter card mirrors OpenGraph on localized routes.

**JsonLd:** localized `description`, `url` (`absoluteUrlForLocale`), and `inLanguage` on `/ru` home.

**Prod smoke:** `npm run verify:prod` — batched curl on `/ru`, `/ru/motion`, `/ru/status`, 404.

**Backend smoke:** `npm run verify:backend` — Convex on `/ru/status` (warn if env missing).

**Messages parity:** `npm run verify:messages` — en.json ↔ ru.json keys (CI).

**Checkout API:** POST `/api/checkout` accepts `locale` (`en`|`ru`) for localized errors.

**Stripe redirects:** success/cancel URLs use `/[locale]?checkout=success` and `/[locale]#pricing` when i18n is on.

**Ship pipeline:** `npm run ship:when-ready` — qa:all → verify:messages → sitemap → deploy → prod smoke.

**Contact form:** `contact.emailLabel`, `contact.leadCount`, `contact.genericError` in messages.

**Loading UI:** `Loading` namespace for route transitions.

| Path | Purpose |
|------|---------|
| `src/lib/i18n/motion-lab-copy.ts` | Motion lab demo types + defaults |
| `src/lib/i18n/build-motion-lab-copy.ts` | `buildMotionLabCopy()` for `[locale]/motion` |

**SEO:** `sitemap.xml` is **dynamic** (`force-dynamic`) — resolves `NEXT_PUBLIC_SITE_URL` / `VERCEL_URL` at request time. Lists each locale URL with `alternates.languages` (en, ru, x-default) when i18n is on.

**Production URL:** `siteConfig.url` uses `NEXT_PUBLIC_SITE_URL`, then `VERCEL_PROJECT_PRODUCTION_URL`, then `VERCEL_URL`, then localhost.

## Locale switcher

`LocaleSwitcher` in site header — visible only when `siteFeatures.i18n`.

## Verify

```bash
NEXT_PUBLIC_I18N=true npm run dev
# http://localhost:3000 → /en
# http://localhost:3000/ru — Russian hero
npm run build
```
