# Как это работает — простыми словами

Этот документ — **для человека, который не хочет разбирать архитектуру**.  
Технический план: [ROADMAP.md](./ROADMAP.md). Практика в Cursor: [HOW-TO-USE.md](./HOW-TO-USE.md).

---

## 1. Что у вас вообще есть

Представьте **строительную фирму**, а не «один готовый сайт».

| Часть | Аналогия | Где лежит |
|-------|----------|-----------|
| **web-motion-starter** | Завод и эталонный дом-выставка | этот репозиторий |
| **Секции** (hero, pricing, faq…) | Готовые комнаты из каталога | `src/components/sections/` |
| **Пресеты** (saas, agency…) | Типовые планировки квартир | `--preset saas` |
| **Скрипт new-web-site** | «Нажми кнопку — получи каркас нового дома» | `~/Projects/scripts/new-web-site.sh` |
| **Cursor + skill site-builder** | Прораб, который знает правила завода | чат агента |
| **Live-сайты** | Построенные дома, которые можно показать | vercel.app ссылки |

Вы **не рисуете каждый сайт с нуля**.  
Вы **заказываете планировку → подставляете тексты и цвет → проверяете → выкладываете**.

---

## 2. Два режима — самое важное правило

### Режим А — «работа на заводе» (template)

Папка: `~/Projects/web-motion-starter`

Делаете, когда нужно **улучшить саму библиотеку**:
- новая страница (privacy, blog…);
- новый вариант hero;
- новый пресет;
- проверки, документация.

Это Roadmap v3 (фазы E–H).

### Режим Б — «стройка для клиента»

Папка: `~/Projects/имя-клиента` (клон с завода)

Делаете, когда нужен **конкретный сайт**:
- тексты, логотип, цвет;
- какие секции включить;
- деплой на Vercel.

**Ошибка новичка:** править клиентский текст прямо в `web-motion-starter`.  
Так вы пачкаете эталон. Клиенту всегда — **своя папка-клон**.

## 3. Как рождается один сайт (сегодня, уже работает)

### Шаг 1. Бриф (что хотим)

**Лучший способ:** файл `brief.json` по схеме [schemas/brief.schema.json](./schemas/brief.schema.json)  
(примеры в `docs/examples/`). Проверка: `npm run verify:brief -- путь.json`.

Или таблица в [SITE-BRIEF.md](./SITE-BRIEF.md) — потом перенести в JSON.

### Шаг 2. Scaffold (собрать каркас)

```bash
cd ~/Projects/web-motion-starter
npm run scaffold:from-brief -- docs/examples/brief-saas.json
```

Или по-старому:

```bash
~/Projects/scripts/new-web-site.sh client-acme --preset saas
```

Что происходит «под капотом»: копируется завод → новая папка `~/Projects/<slug>` с уже готовыми секциями.

### Шаг 3. Открыть клон в Cursor

Сказать агенту: работай в `~/Projects/client-acme` (не в starter).

Промпт примерно такой:

> Use site-builder skill. Edit content/site.json and brand. Keep sections from catalog. QA then ship.

### Шаг 4. Наполнить

| Что меняете | Где |
|-------------|-----|
| Заголовки, меню, порядок блоков | `content/site.json` |
| Цвет бренда | `src/lib/theme/brand-palette.css` |
| Русский/английский текст (если i18n) | `messages/ru.json`, `messages/en.json` |

**Не нужно** переписывать React-компоненты ради смены фразы «Начать» → «Попробовать».

### Шаг 5. Посмотреть локально

```bash
cd ~/Projects/client-acme
npm install
npm run dev
```

Открыть http://localhost:3000

### Шаг 6. Проверить и выложить

```bash
npm run qa:all        # код не сломан + скриншоты
npm run ship:client   # выкладка на Vercel
```

Готово: есть публичная ссылка.

**Ориентир по времени:** простой лендинг — около 15–30 минут, если тексты уже есть.

## 4. Что уже умеет «завод» (фазы 1–42)

Простыми словами:

1. **Красивый лендинг** с анимациями (не «голый HTML»).
2. **Сборка из блоков** — включил pricing, выключил team.
3. **Два языка** (en/ru) на эталоне.
4. **Форма заявок** (если подключить Convex).
5. **Оплата тарифов** (если подключить Stripe).
6. **Проверки после деплоя** — скрипты сами смотрят, что на проде русский текст на месте.
7. **Обновление клиентов** с завода без затирания их текстов (`sync:clone`).

Это уже **не черновик**. Это рабочий конвейер для **одностраничных** сайтов (главная + служебные motion/status).

## 5. Зачем Roadmap v3 — большими мазками

Сейчас завод хорошо делает **одну главную страницу**.

Клиентам в жизни часто нужно ещё:

- политика и оферта;
- «О нас»;
- кейс / портфолио-страница;
- простой блог;
- другой вид hero (не всегда один и тот же);
- чтобы агент в Cursor меньше «фантазировал» и шёл по брифу.

v3 — это **четыре больших улучшения завода**, по 5 дней каждое.

### Фаза E (дни 1–5) — «не только главная»

**Сейчас:** дом = гостиная (home).  
**Будет:** можно добавить коридор legal, кабинет about, витрину case, полочку blog — **по желанию**, не обязательно всем.

Аналогия: в каталоге стройфирмы появились готовые «доп. комнаты». При заказе ставите галочки: privacy ✓, about ✓, blog ✗.

### Фаза F (дни 6–10) — «не один дизайн на всех»

**Сейчас:** hero почти всегда одного вида, цвет крутите hue.  
**Будет:**

- варианты комнаты: hero «обычный» / hero «редакционный»;
- pricing «карточки» / «сравнительная таблица»;
- **theme packs** — готовые настроения оформления (не только «крутилка цвета»).

Аналогия: одна планировка, но отделка «минимализм» или «студия» или «продуктовый тёмный».

### Фаза G (дни 11–15) — «агент как умный прораб» ✅

**Уже в коде (v1.0.0):**

1. `brief.json` → `npm run verify:brief`
2. `npm run scaffold:from-brief -- brief.json`
3. Автоматически: `copy:apply` (hero/бренд в messages)
4. Агент обязан пройти **QA gates** — без этого нельзя сказать «готово»
5. Версия завода **1.0.0** + [CHANGELOG](../CHANGELOG.md) + [TEMPLATE-SYNC](./TEMPLATE-SYNC.md)

Аналогия: заказ-наряд → завод режет детали → прораб не сдаёт дом без проверки.

### Фаза H (дни 16–20) — «после сдачи ключей»

**Сейчас:** сайт выложен — дальше вручную.  
**Будет:**

- счётчик посещений (analytics) одной настройкой;
- письмо вам, когда кто-то оставил email;
- CMS на `/status` без случайной порчи прода;
- пресет `launch` «быстрый запуск продукта»;
- короткая «продающая» памятка, что вы вообще продаёте клиенту.

### Фаза A\* (опционально) — «подключить воду и электричество»

Convex, свой домен, Stripe — это **не улучшение завода**, а **подключение ваших аккаунтов** к уже построенному дому.  
Можно делать в любой момент. **Не мешает** строить фазы E–G.

## 6. Один день из плана — как выглядит на практике

Возьмём **День 2 v3** (legal pages) максимально приземлённо.

1. Утром открываете Cursor в папке **web-motion-starter** (режим А).
2. Пишете:  
   `Implement Roadmap v3 Day 2 exactly as in docs/ROADMAP.md`
3. Агент добавляет страницы «Политика» и «Условия» на EN и RU.
4. Вы открываете в браузере `/ru/privacy` — читается по-русски.
5. В конце дня: коммит, деплой эталона, галочка «день 2 готов».

На **день 12** уже в режиме Б:

1. Кладёте brief клиента в JSON.
2. `npm run scaffold:from-brief -- brief.json`
3. Агент в клоне допиливаете тексты.
4. `qa` → `ship` → ссылка клиенту.

## 8. Частые вопросы чайника

**Это библиотека npm вроде React?**  
Нет. Это **шаблон + конвейер + правила для Cursor**. Вы клонируете проект, а не `npm install web-motion-starter` как пакет UI.

**Мне нужно знать Next.js?**  
Для сдачи клиентского сайта — минимально. Меняете JSON и цвет, агент пишет код. Для улучшения завода (v3) — лучше понимать структуру папок, но можно вести агента по ROADMAP «день N».

**Сломаю ли я клиентские сайты, если улучшаю завод?**  
Нет, если работаете в starter и потом осторожно `sync:clone`. Тексты клиента не должны затираться.

**Обязателен русский?**  
Нет. i18n включается флагом. Эталон уже умеет `/ru`.

**Обязательны Convex и Stripe?**  
Нет. Без них сайт — красивая статика + форма может быть заглушкой. Подключаете, когда нужны живые заявки и оплата.

**С чего открыть в браузере, чтобы «пощупать»?**  
https://web-motion-starter.vercel.app/ru — эталон.  
https://site-ref-saas.vercel.app — пресет SaaS.  
https://client-flowstack.vercel.app — пример клиентской сдачи.

## 9. Одна картинка в голове

```
   БРИФ (что нужно человеку)
        │
        ▼
   SCAFFOLD (копия завода → папка клиента)
        │
        ▼
   НАПОЛНЕНИЕ (тексты + цвет; блоки из каталога)
        │
        ▼
   ПРОВЕРКА (qa / verify)
        │
        ▼
   ДЕПЛОЙ (Vercel) → ссылка клиенту
```

**Roadmap v3** не ломает эту схему.  
Он делает так, чтобы:

- блоков и **страниц** было больше;
- внешний вид **гибче**;
- Cursor **меньше импровизировал**;
- после сдачи было **проще жить**.

## 10. Что сделать прямо сейчас

1. Открыть эталон: https://web-motion-starter.vercel.app/ru  
2. Прочитать этот файл ещё раз до раздела 7.  
3. Выбрать горизонт: **3 дня** или **фаза E**.  
4. В Cursor (в `web-motion-starter`) написать:

```
Сделай День N из docs/ROADMAP.md точно по таблице дня.
Объясняй простыми словами, что меняешь.
В конце — как проверить в браузере.
```

Подставьте N = 1 (или 11, если хотите сразу «умный прораб»).
